Lua scripting lets you change game *behavior* at runtime: hook into game functions, read and write live
objects, add commands, and react to events - all without cooking assets. It runs on
**[UE4SS](/frameworks/ue4ss/)**.

## In this section

- **[Lua modding in Palworld](/lua-scripting/lua-modding-in-palworld/)** - getting a UE4SS Lua mod
  loading, the mod folder layout, and the basics of hooking and calling game functions.

Lua is the right tool when you want logic and interactivity. If you only need to change values, a
[data mod](/data-modding/) is simpler; if you need new Blueprint content, see the
[Palworld Modding Kit](/frameworks/palworld-modding-kit/).
